Exploring Cohon Calibration

2026-03-02

1. Introduction

Boundary penalties are used in spatial conservation planning to reduce fragmentation. In prioritizr, add_boundary_penalties() encourages solutions to form compact clusters of planning units rather than scattered selections. Compact solutions are often easier to manage and may better support ecological processes, but they usually come at a higher cost.

A common challenge is that the boundary penalty is often set arbitrarily. If the penalty is too small, solutions remain fragmented. If it is too large, solutions become overly compact and can become very expensive. This makes it hard to judge whether the implied trade-off between cost and compactness is reasonable.

The function calibrate_cohon_penalty() in prioritizr provides a more systematic approach. It applies Cohon-style calibration to estimate a boundary penalty that aims to balance cost and compactness, using two extreme solutions (see Section 2: A brief note on Cohon-style calibration).

In this vignette, we use the Tasmania example dataset to explore how this calibration behaves under simple, controlled changes to the problem. Specifically, we ask:

  1. How does the calibrated boundary penalty change as we change the conservation target?
  2. Do target–penalty patterns stay similar when feature complexity is reduced (and when changes are applied in reverse order)?
  3. How does the calibrated boundary penalty change as we change the number of features?
  4. Is the feature–penalty relationship consistent across different feature compositions (and in reverse order)?
  5. What are the runtime implications of Cohon-style calibration?

The goal is not to find a single “correct” penalty. Instead, we aim to understand how sensitive the calibration is to problem structure, and what that means for decision-support tools such as spatialplanr and shinyplanr.

2. A brief note on Cohon-style calibration

Very briefly, calibrate_cohon_penalty():

  1. Solves an “ignore boundary” problem (penalty ≈ 0) to find a very cheap but fragmented solution.

  2. Solves a “boundary-only” problem (very large penalty) to find a very compact but otherwise expensive solution.

  3. Uses these two “anchor” solutions (costs and boundary lengths) to compute a boundary penalty that trades off cost and boundary length in a more balanced way.

  4. Returns this penalty so we can re-solve the problem with a single calibrated value.

The calibrated penalty therefore depends on how fragmented the cheap solution is and how expensive the compact solution needs to be. We therefore explore how changing targets and reducing the number of features affects the calibrated penalty.

4. Helper functions

To keep the analysis readable, we create helper functions to summarise solution characteristics and standardise how problems are built, calibrated, and solved across scenarios.

4.1 Fragmentation and cost metrics

Each solution is summarised using a small set of interpretable metrics, including selected area, patch structure, boundary length, cost, and solver runtime.

summarise_solution <- function(label,
                               solution_sf,
                               solve_time_sec,
                               min_patch_size   = NULL,
                               boundary_penalty = 0) {

  # Check inputs
  if (!"solution_1" %in% names(solution_sf)) {
    stop("summarise_solution(): column 'solution_1' not found in solution_sf")
  }
  if (!"cost" %in% names(solution_sf)) {
    stop("summarise_solution(): column 'cost' not found in solution_sf")
  }

  # Select PUs
  sel <- solution_sf %>%
    dplyr::filter(.data$solution_1 == 1)

  n_selected <- sel %>%
    nrow()

  # Return empty summary
  if (n_selected == 0) {
    return(
      tibble::tibble(
        method             = label,
        n_selected         = 0L,
        total_area_km2     = 0,
        n_patches          = 0L,
        valid_patches      = NA_integer_,
        median_patch_km2   = NA_real_,
        pu_cost_total      = 0,
        pu_cost_mean       = NA_real_,
        boundary_length_km = 0,
        boundary_cost      = 0,
        total_cost         = 0,
        solver_time_sec    = solve_time_sec
      )
    )
  }

  # Compute area
  pu_areas_m2 <- sel %>%
    sf::st_area() %>%
    as.numeric()

  total_area_km2 <- pu_areas_m2 %>%
    sum() %>%
    `/`(1e6)

  # Build patches
  patches <- sel %>%
    sf::st_union() %>%
    sf::st_cast("POLYGON")

  patch_areas_m2 <- patches %>%
    sf::st_area() %>%
    as.numeric()

  n_patches <- patch_areas_m2 %>%
    length()

  median_patch_km2 <- patch_areas_m2 %>%
    stats::median() %>%
    `/`(1e6)

  # Count valid patches
  valid_patches <- if (!is.null(min_patch_size)) {
    min_patch_size_m2 <- min_patch_size %>%
      as.numeric()
    patch_areas_m2 %>%
      `>=`(min_patch_size_m2) %>%
      sum()
  } else {
    NA_integer_
  }

  # Measure boundary
  boundary_length_km <- patches %>%
    sf::st_boundary() %>%
    sf::st_cast("MULTILINESTRING") %>%
    sf::st_length() %>%
    as.numeric() %>%
    sum() %>%
    `/`(1000)

  # Summarise costs
  pu_cost_total <- sel %>%
    dplyr::pull(.data$cost) %>%
    sum(na.rm = TRUE)

  pu_cost_mean <- sel %>%
    dplyr::pull(.data$cost) %>%
    mean(na.rm = TRUE)

  boundary_cost <- boundary_length_km * boundary_penalty
  total_cost    <- pu_cost_total + boundary_cost

  # Return summary
  tibble::tibble(
    method             = label,
    n_selected         = n_selected,
    total_area_km2     = total_area_km2,
    n_patches          = n_patches,
    valid_patches      = valid_patches,
    median_patch_km2   = median_patch_km2,
    pu_cost_total      = pu_cost_total,
    pu_cost_mean       = pu_cost_mean,
    boundary_length_km = boundary_length_km,
    boundary_cost      = boundary_cost,
    total_cost         = total_cost,
    solver_time_sec    = solve_time_sec
  )
}

4.2 Building and calibrating problems

Next, we define a small helper to: 1. build a prioritizr problem for a given target and a chosen set of features, 2. add a temporary boundary penalty, 3. run calibrate_cohon_penalty() to find a calibrated boundary penalty, 4. solve the problem with the calibrated penalty, and 5. return a tidy row with the penalty, timings, and solution metrics.

# Create function for prioritisation
run_cohon_experiment <- function(label, target, feature_names) {

  prob <- prioritizr::problem(
    x           = tas,
    features    = feature_names,
    cost_column = "cost"
  ) %>%
    prioritizr::add_min_set_objective() %>%
    prioritizr::add_relative_targets(target) %>%
    prioritizr::add_binary_decisions() %>%
    prioritizr::add_rsymphony_solver(verbose = TRUE)

  calib_time <- system.time({
    cohon_penalty <- prioritizr::calibrate_cohon_penalty(
      prob %>% prioritizr::add_boundary_penalties(penalty = 1e-4, edge_factor = 1),
      approx  = TRUE,
      verbose = TRUE
    )
  })

  cohon_penalty <- as.numeric(cohon_penalty)
  if (length(cohon_penalty) != 1 || !is.finite(cohon_penalty)) {
    stop("calibrate_cohon_penalty() did not return a single finite numeric penalty.")
  }

  prob_final <- prob %>%
    prioritizr::add_boundary_penalties(penalty = cohon_penalty, edge_factor = 1)

  solve_time <- system.time({
    solution <- solution <- solve(prob_final, force = TRUE)
  })

  metrics <- summarise_solution(
    label            = label,
    solution_sf      = solution,
    solve_time_sec   = solve_time[["elapsed"]],
    min_patch_size   = NULL,
    boundary_penalty = cohon_penalty
  )
  
  # Create table
  dplyr::mutate(
    metrics,
    scenario       = label,
    n_features     = length(feature_names),
    target         = target,
    cohon_penalty  = cohon_penalty,
    calib_time_sec = calib_time[["elapsed"]],
    total_time_sec = calib_time[["elapsed"]] + solve_time[["elapsed"]]
  )
}
